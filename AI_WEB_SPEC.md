# 筆電挑選器網頁規格書

## 1. 專案目標

建立一個繁體中文的筆電挑選網站，資料完全來自資料夾內的 Excel 轉出資料與對應圖片。

網站用途：
- 讓使用者快速找到合適的筆電機型
- 支援以預算、用途、硬體規格篩選
- 顯示教育價 / 專案價與市價比較
- 支援多台機型比較
- 自動顯示機型圖片，且依 Excel 的機型名稱對應

## 2. 資料來源

### 2.1 Excel

Excel 內包含限定機型與價格資料。

規則：
- 只可顯示 Excel 內出現的機型
- 不可額外加入 Excel 沒有的機型
- 資料更新時，以 Excel 為主要資料來源
- 教育價 / 專案價是主價格
- 市價要保留，用於比較與營造折扣感

### 2.2 圖片

圖片放在專案資料夾中，且圖片檔名或資料夾名稱需可與 Excel 的機型名稱對應。

配對規則：
- 以 `model` 或完整機型名稱作為圖片對應依據
- 優先使用能精準對到機型名稱的圖片
- 每台機型可以有多張圖片，用於輪播
- 若某機型找不到圖片，需保留預設備用圖

圖片要求：
- 使用本機型對應照片，不使用無關的示意圖
- 依機型名稱自動配對，不要手動逐台寫死
- 建議輸出為適合網頁載入的壓縮格式

## 3. 資料欄位

網站至少要使用以下欄位：

- `model`：機型代號
- `title`：完整標題
- `family`：系列名稱
- `cpu`：CPU
- `memory`：RAM
- `storage`：SSD
- `gpu`：顯示卡
- `display`：螢幕規格
- `weight`：重量
- `warranty`：保固
- `bundle`：配件或內容
- `marketPrice`：市價
- `eduPrice`：教育價 / 專案價
- `discount`：目前最高折扣
- `discountRate`：折扣率
- `highlights`：重點特色
- `purposes`：用途標籤
- `image`：主圖
- `imageKind`：圖片類型
- `screenSize`：螢幕尺寸
- `ramGB`：RAM 數值
- `storageGB`：SSD 容量
- `rtx`：是否為 RTX 系列
- `oled`：是否 OLED
- `ai`：是否 AI 機種
- `gpuTier`：顯示卡等級
- `performance`：效能分數
- `valueScore`：性價比分數
- `searchText`：搜尋字串

## 4. 首頁需求

首頁要直接是可操作的挑選頁，不要做成純介紹頁。

### 4.1 首屏內容

- 標題清楚表達這是筆電挑選器
- 一進站就能看到推薦機型
- 推薦機型以橫式輪播呈現，避免畫面跳動
- 手機版可不輪播，直接顯示教育價最低的一組推薦機型即可

### 4.2 推薦機型規則

- 桌機版：顯示輪播
- 輪播內容預設為教育價介於 `25000 ~ 30000` 的機型
- 若該區間不足，才依規則補上接近區間的機型
- 推薦卡片要顯示：
  - 圖片
  - 機型
  - 教育價
  - 市價
  - 目前最高折扣

## 5. 篩選規則

網站主要用下拉選單篩選，讓使用者快速縮小範圍。

### 5.1 篩選條件

以下欄位都要提供下拉選單：

- 預算
- 用途
- CPU
- RAM
- SSD
- LCD
- 顯示卡

### 5.2 CPU 分類

CPU 下拉選單要再細分，至少要看得出以下系列：

- Intel Core i5
- Intel Core i7
- Intel Core i9
- Intel Core Ultra
- AMD Ryzen 5
- AMD Ryzen 7
- AMD Ryzen 9
- AMD Ryzen AI

若同系列下還有細分類，建議依實際型號再分組，但不要把分類做得太粗。

### 5.3 RAM 分類

RAM 只用容量分類，不要出現其他混雜分類。

要能辨識這類項目：
- `8G`
- `16G`
- `16G*2`
- `32G`
- `32G*2`
- `64G`

### 5.4 SSD 分類

SSD 只用容量分類，不要用其他描述方式。

建議選項：
- `512G`
- `1T`
- `2T`
- 其他容量可依實際資料延伸

### 5.5 LCD 分類

LCD / 螢幕尺寸以尺寸區分。

要保留 13 吋機型，不可因資料整理而消失。

建議選項：
- `13 吋`
- `14 吋`
- `15 吋`
- `16 吋`
- `其他尺寸`

### 5.6 顯示卡分類

顯示卡要能讓使用者看懂差異，至少區分：

- 內顯
- RTX 4050
- RTX 4060
- RTX 4070
- RTX 5060
- RTX 5070
- 其他等級

## 6. 顯示規則

### 6.1 價格顯示

預設狀態：
- 預設不顯示教育價
- 教育價欄位要以「報價請洽服務人員」呈現

切換方式：
- 在最上方的 `EXCEL` 字樣加入隱藏切換功能
- 使用者點選 `EXCEL` 之後，才切換成顯示教育價 / 不顯示教育價
- 不要直接顯示「顯示教育價」或「不顯示教育價」這類明顯文字
- 切換方式要隱藏，不要教學式露出

其他價格規則：
- `目前最高折扣` 要保留顯示
- 市價要持續顯示
- 教育價與市價要一起出現，讓價格對比明顯

### 6.2 機型卡片

每台筆電卡片應該顯示：
- 圖片
- 機型名稱
- CPU
- RAM
- SSD
- LCD
- 顯示卡
- 教育價或報價提示
- 市價
- 目前最高折扣
- 勾選比較

### 6.3 圖片輪播

- 每台機型圖片要可以輪播
- 圖片來源優先使用資料夾中與機型名稱對應的照片
- 輪播時不要讓版面跳動
- 推薦區與機型卡片都要盡量維持穩定高度

## 7. 多機比較

### 7.1 勾選比較

每台卡片下方要有比較勾選框。

### 7.2 比較方式

- 使用者可以勾選多台
- 按下比較後，進入多機比較畫面
- 多機比較要以直式比較呈現
- 比較區最上方先顯示圖檔
- 接著依欄位逐項比較

### 7.3 比較欄位

比較至少要顯示：
- 圖片
- 機型
- CPU
- RAM
- SSD
- LCD
- 顯示卡
- 教育價
- 市價
- 目前最高折扣
- 重量
- 保固

## 8. 版面原則

- 第一屏要可直接操作
- 不要做成單純形象頁
- 不要把重要功能藏在深層頁面
- 桌機版重視比較效率
- 手機版重視快速瀏覽與少量資訊
- 色彩要清楚、有層次，避免過度單一色系
- 圖片要真實對應機型，不要亂配

## 9. 更新流程

如果之後要更新資料，請用以下方式：

1. 把新的 Excel 放進專案資料夾
2. 把新的機型照片放進同一資料夾或子資料夾
3. 確保照片名稱能對應機型名稱
4. 重新產生資料表與圖片對照
5. 重新部署網站

更新時的注意事項：
- 不要新增 Excel 以外的機種
- 不要讓配對錯誤圖片混入正確機型
- 若某些機型找不到照片，先保留備用圖，不要中斷整個網站

## 10. 給其他 AI 的實作要求

請其他 AI 生成網站時，務必遵守以下原則：

- 以 Excel 內的限定機型為唯一資料來源
- 使用機型名稱自動配圖
- 預設隱藏教育價
- 使用者點 `EXCEL` 才切換教育價顯示
- 保留市價與目前最高折扣
- 提供 CPU / RAM / SSD / LCD / 顯示卡下拉篩選
- 提供多機勾選比較
- 推薦區桌機用橫式輪播，手機版可不輪播
- 多機比較採直式排列，圖片置頂
